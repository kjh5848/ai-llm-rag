# AI 업무 비서(RAG + MCP) 집필 문체 가이드

본 문서는 사내 지식 엔진 구축 시리즈의 원고 집필 시 적용할 문장 스타일과 어조를 정의합니다. 독자가 기술적 내용을 명확하게 습득하고 프로젝트를 완수할 수 있도록 돕는 것이 가이드의 목적입니다.

---

## 1. 지향하는 핵심 어조 (Tone of Voice)

### 1.1 신뢰 기반의 실천적 어조 (Practical & Authoritative)
- 기술적 사실에 근거하여 명확하고 단정적인 문장을 사용합니다.
- "~인 것 같습니다"와 같은 추측성 표현을 배제하고, 시스템의 동작 원리를 확신 있게 전달합니다.

### 1.2 친절한 사수의 시선 (Mentoring Style)
- 독자를 함께 프로젝트를 수행하는 동료로 대우하며 가이드를 제공합니다.
- 복잡한 개념은 비유와 도식을 적극적으로 활용하여 심리적 장벽을 낮춥니다.

---

## 2. 문체 규칙 (Grammar & Style Rules)

### 2.1 하십시오체의 일관된 유지
- 본문 전체에 걸쳐 정중하고 절제된 **'하십시오체'**(~합니다, ~입니다)를 사용합니다.
- 이는 전문 기술 서적으로서의 권위를 유지하면서도 독자에게 신뢰감을 줍니다.

### 2.2 능동형 문장 중심의 서술
- 주체가 명확한 능동형 문장을 사용하여 텍스트의 활력을 높입니다.
- 예: "데이터가 수집됩니다" → "시스템이 데이터를 수집합니다" 혹은 "데이터를 수집합니다"

### 2.3 전문 용어의 엄밀한 처리
- 한국어 용어를 우선하되, 명확한 개념 전달을 위해 영문(English)을 병기합니다.
- 예: 검색 증강 생성(RAG, Retrieval-Augmented Generation)

---

## 3. 집필 전략: 코드 설명 vs 시스템 이해

### 3.1 구조적 이해 우선 (PART 1~2)
- 이 책의 기초가 되는 사내 시스템(DB, CRUD 사이트)은 코딩 실습이 아닌 **Git Clone**을 통해 준비합니다.
- 따라서 이 구간에서는 코드 한 줄 한 줄을 설명하기보다, 전체적인 **데이터 구조와 아키텍처**를 이해시키는 데 집중합니다.

### 3.2 실전 구현 (PART 3 이후)
- RAG 구축부터는 독자가 직접 환경을 구축하고 파이프라인을 설계하는 과정을 상세히 서술합니다.
- 각 단계의 목적(Why)을 명시하고, 그 결과가 시스템 전체에 어떤 영향을 주는지 설명합니다.

---

## 4. 문장 교정 예시

- **Before**: 이제 파이썬 가상환경을 하나 만들어보도록 할게요.
- **After**: 프로젝트의 독립적 관리를 위해 Python 가상환경을 생성합니다.
- **Before**: 이미지가 안보이는건 경로 문제인것 같아요.
- **After**: 이미지 표시가 되지 않는 현상은 대부분 파일 경로 참조 오류로 인해 발생합니다.
