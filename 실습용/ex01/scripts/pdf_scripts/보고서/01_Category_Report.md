# [Category 01] 다단 편집(Multi-Column) RAG 최적화 보고서

본 보고서는 **다단 편집(Multi-Column)** 문서에서 발생하는 주요 파싱 문제와, 이를 해결하기 위한 **RAG 최적화 전처리 전략**의 전후 비교를 다룹니다.

---

## 1. Case 1: 폰트 찢어짐 (Word Shredding)
**문단 정렬(Justify)**로 인해 글자 사이 공백이 멀어져, 단어가 자소 단위나 음절 단위로 쪼개지는 현상입니다.

| 구분 | 기본 파서 (Default) | RAG 최적화 (Optimized) |
| :--- | :--- | :--- |
| **문제점** | `대 한 민 국` 처럼 글자가 흩어져 **키워드 검색 불가**. | `x_tolerance` 임계값을 조정하여 시각적 공백 무시. |

| **결과** | 토큰화 불가능 (Trash Data) | **"대한민국"** 으로 온전한 단어(Token) 복원. |

### 🛠️ 기술 분석 (Technical Analysis)

| 구분 | 내용 (Content) |
| :--- | :--- |
| **Library** | `pdfplumber` (Python 3.10+) |
| **Failure Reason** | `extract_text()` (Default) -> `x_tolerance=3` (기본값) <br> : 시각적 공백을 너무 민감하게 인식하여 단어를 자소 단위로 쪼갬. |
| **Success Reason** | `extract_text(x_tolerance=5)` <br> : 허용 오차를 늘려 흩어진 자소를 하나의 단어로 합침. |

### 🔑 Key Code (Solution)
```python
# pdfplumber의 x_tolerance 옵션을 조정하여 찢어진 단어 봉합
with pdfplumber.open(input_pdf) as pdf:
    for page in pdf.pages:
        # x_tolerance=5: 글자 사이 간격이 5pt 이내면 같은 단어로 취급
        text = page.extract_text(x_tolerance=5)
```

## 2. Case 2: 텍스트 스티칭 (Text Stitching)
다단 편집된 문서의 **좌측 단과 우측 단이 한 줄로 병합**되어, 문맥이 뒤섞이는 현상입니다.

| 구분 | 기본 파서 (Default) | RAG 최적화 (Optimized) |
| :--- | :--- | :--- |
| **문제점** | `좌측문장 우측문장` 형태로 합쳐져 **의미 파악 불가**. | **페이지 중앙(x축)**을 기준으로 좌/우 영역 분할(Crop). |

| **결과** | 할루시네이션(Hallucination) 유발. | **올바른 읽기 순서(Reading Order)** 확립. |

### 🛠️ 기술 분석 (Technical Analysis)

| 구분 | 내용 (Content) |
| :--- | :--- |
| **Library** | `pdfplumber` |
| **Failure Reason** | `extract_text()` (Default) <br> : 레이아웃 정보(Column)를 무시하고 Y좌표(줄) 단위로 텍스트를 읽어, 왼쪽/오른쪽 단을 한 줄로 합쳐버림. |
| **Success Reason** | **Geometric Cropping (공간 분할)** <br> : 페이지를 물리적으로 반으로 자른(`crop`) 후, 왼쪽 -> 오른쪽 순서로 추출하여 문맥 보존. |

### 🔑 Key Code (Solution)
```python
# 페이지 폭(width)을 구해 정확히 절반(mid)으로 나눔
width = page.width
mid = width / 2

# 왼쪽 단 (Left Column) -> 좌표 기반 Cropping
left_bbox = (0, 0, mid, height)
left_text = page.crop(left_bbox).extract_text()

# 오른쪽 단 (Right Column)
right_bbox = (mid, 0, width, height)
right_text = page.crop(right_bbox).extract_text()
```

## 3. Case 3: 내포된 표/박스 (Nested Elements)

| 구분 | 기본 파서 (Default) | RAG 최적화 (Optimized) |
| :--- | :--- | :--- |
| **문제점** | 박스 내부 텍스트가 본문 중간에 삽입됨 → **문맥 단절**. | `Rect` 객체를 감지하여 **독립적인 영역(Zone)**으로 추출. |

| **결과** | 주제가 다른 정보 혼재. | **본문과 메타데이터(수치 등) 분리** 성공. |

### 🛠️ 기술 분석 (Technical Analysis)

| 구분 | 내용 (Content) |
| :--- | :--- |
