# 📖 Case 05: 시각 정보 (Vision Analysis)

## 1. 개요 (Summary)
- **난이도 (Difficulty)**: 중 (Medium)
- **주요 기술 (Key Tech)**: Multimodal LLM (GPT-4o, Claude 3.5 Sonnet)
- **핵심 개념 (Core Concept)**: 차트나 다이어그램은 "픽셀"이 아니라 **"정보(Information)"** 입니다. AI에게 단순한 묘사가 아닌, **"결과값 추출"**을 지시해야 합니다.

## 2. 케이스 분석 (Case Analysis)

### 🔹 Case 1: 차트 (Chart)
- **입력 (Input)**: `05_case1_차트.pdf`
- **문제 (Problem)**: **"문맥의 부재"**. 단순 텍스트 추출(OCR)로는 수치가 있어도, 그것이 매출인지 이익인지, 어떤 연도에 해당하는지 알 수 없습니다.
- **해결 (Solution)**: **Structured Data Extraction**. VLM에게 "X축은 연도, Y축은 매출액"이라고 정의해주고, 결과값을 JSON 형식으로 추출합니다.

### 🔹 Case 2: 플로우차트 (Flowchart)
- **입력 (Input)**: `05_case2_플로우차트.pdf`
- **문제 (Problem)**: **"비선형적 구조"**. 순서도나 시스템 아키텍처는 위에서 아래로 읽는 텍스트와 달리, 화살표를 따라 논리가 점프하거나 순환합니다.
- **해결 (Solution)**: **Logical Path Tracing**. "시작점(Start Node)에서 출발하여 끝점(End Node)까지의 모든 경로를 텍스트로 서술하라"고 지시하여 프로세스를 선형화합니다.

### 🔹 Case 3: 복합 범례 (Complex Legend)
- **입력 (Input)**: `05_case3_복합범례.pdf`
- **문제 (Problem)**: **"색상과 매핑"**. 10개 이상의 꺾은선 그래프가 겹쳐 있을 때, 선의 색상과 범례(Legend)를 정확히 매칭하는 것은 일반 OCR로는 불가능합니다.
- **해결 (Solution)**: **Visual Cross-Referencing**. "빨간색 실선은 Apple, 파란색 점선은 Samsung"과 같이 시각적 특징을 기반으로 데이터를 분류하도록 유도합니다.

## 3. RAG 구축 전략 (RAG Strategy)

시각 정보(Vision) 처리의 핵심은 **"픽셀의 텍스트화(Textualization of Pixels)"** 입니다. 이미지 속에 갇힌 데이터를 LLM이 검색하고 추론할 수 있도록 논리적인 텍스트나 정형 데이터(JSON/Markdown)로 변환하는 과정이 훌륭한 RAG 성능을 결정합니다.

| 문제 상황 | RAG에 미치는 영향 (Risk) | 해결 전략 (Solution) |
| :--- | :--- | :--- |
| **통계 차트 (Case 1)** | 차트 속 숫자가 문맥 없이 나열되면, 특정 연도의 매출액 같은 분석적 질문에 오답을 냅니다. | **구조적 데이터 추출 (JSON/CSV)**: VLM에게 축(Axis) 정의를 주고 데이터를 정형화된 형식으로 뽑아내 검색 엔진에 주입합니다. |
| **플로우차트 (Case 2)** | 화살표로 이어진 비선형적 로직을 단순 OCR로 읽으면 선후 관계가 뒤섞여 프로세스 이해에 실패합니다. | **논리 경로 서술 (Path Tracing)**: 시작부터 끝까지의 흐름을 텍스트로 풀어 써서 순서 정보를 보존합니다. |
| **복합 범례 (Case 3)** | 여러 선이 겹친 그래프에서 색상과 데이터의 매칭이 깨지면 전혀 엉뚱한 브랜드의 수치를 인용하게 됩니다. | **시각적 상호 참조 (Cross-Referencing)**: 색상, 선 모양 등 시각적 특징을 명시하여 데이터를 정확히 분류하도록 유도합니다. |

> **본 실습의 핵심**:
> - **의미론적 설명**: AI에게 단순히 "그림을 설명해"라고 하지 말고, "이 그림이 RAG 시스템에서 어떤 질문에 대답하기 위해 필요한 정보인가?"를 먼저 고민하고 추출 지시(Prompt)를 설계해야 합니다.
