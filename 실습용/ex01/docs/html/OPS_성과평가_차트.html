제6장. 데이터 기반 성과 평가 (Data-Driven Review)

기존의 정성적 평가 방식은 평가자의 주관이 개입될 여지가 많았습니다. 이에 우리 회사는 2026년부터 협업 툴(GitHub, Jira, Confluence)에서 추출된 데이터를 기반으로 하는 '객관적 성과 지표'를 도입하였습니다.

각 항목별 가중치는 직군에 따라 다르며, 개발 직군의 경우 소스 코드 기여도와 동료 간의 코드 리뷰 활동이 핵심 평가 요소로 작용합니다. 상세한 항목별 점수 현황은 아래 [Figure 6-1]을 참조하십시오.

📊 분석 코멘트 (Analysis)
위 차트에서 볼 수 있듯이, 개발팀은 개인의 코딩 양(Commits)보다는 팀 전체의 코드 품질을 높이는 활동에 더 높은 가중치를 부여받고 있습니다. 이는 협업을 중시하는 회사의 핵심 가치와 일치합니다. 반면, 기술 블로그 작성(Tech Blog) 참여율은 상대적으로 저조하여 내년도 개선이 필요합니다.

평가 결과에 따라 S, A, B, C 등급이 부여되며, 상위 20% 인재에게는 연봉 인상 외에도 스톡옵션(Stock Option) 추가 부여 혜택이 제공됩니다.
