# 🤖 [Phase 1] 순수 RAG 에이전트 테스트 리스트

이 문서는 **`ex01`** 실습 전용입니다. 여기서는 **실시간 DB(SQL) 조회 도구가 제거**되었으며, 오직 `docs/` 폴더의 문서 검색(RAG)만 가능합니다.

질문을 통해 RAG의 성능과 DB 연동이 없을 때의 한계를 확인해 보세요.

---

## 1. 성공 케이스 (비정형 문서 검색)

사내 보안 지침이나 휴가 규정 등 **문서에 명시된 내용**은 정확히 찾아 답변합니다.

- **질문 1**: "우리 회사의 연차 부여 기준에 대해 설명해줘."
- **질문 2**: "사내 보안 사고 예방을 위한 핵심 지침 3가지가 뭐야?"
- **질문 3**: "복리후생 제도에는 어떤 것들이 있는지 알려줘."

---

## 2. 한계 확인 케이스 (정형 데이터 조회 실패)

**DB 조회가 필요한 질문**입니다. `ex01` 에이전트는 DB 접근 권한(도구)이 없으므로 답변하지 못하거나 문서 기반의 추측만 하게 됩니다.

- **질문 1**: "김철수님의 남은 휴가 일수는 몇 일인가요?"
  - _예상 결과_: "문서에서 정보를 찾을 수 없습니다." 또는 정책상의 기본 휴가일수만 언급함.
- **질문 2**: "현재 영업팀에 소속된 직원이 총 몇 명이지?"
  - _예상 결과_: 실시간 인원수를 대답하지 못함.
- **질문 3**: "최근 우리 부서의 매출 통계를 요약해줘."
  - _예상 결과_: 매출 데이터를 조회할 수 없다는 안내 출력.

---

## 💡 학습 포인트

1. **RAG의 강점**: 규정이나 매뉴얼처럼 텍스트로 된 방대한 지식을 찾는 데 매우 효율적입니다.
2. **RAG의 약점**: 특정 개인의 실시간 정보, 통계 데이터 등 **숫자와 사실 위주의 데이터**는 문서 검색만으로 답변하기 어렵습니다.
