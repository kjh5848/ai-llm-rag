# 🤖 메타코딩 AI 비서 테스트 질문 리스트

이 문서는 시스템의 하이브리드 검색(RAG + SQL) 성능을 검증하기 위한 테스트셋입니다. 질문을 복사하여 채팅창에 입력해 보세요.

---

## 1. 정형 데이터 검증 (SQL 검색)

데이터베이스에 저장된 직원 정보와 매출 데이터를 정확히 불러오는지 확인합니다.

- **직원 정보**: "김주혁님의 소속 부서와 이메일 주소를 알려줘."
- **조직 현황**: "우리 회사 영업팀에는 총 몇 명의 직원이 근무하고 있나요?"
- **매출 조회**: "최근 영업팀에서 발생한 매출 내역을 요약해줘."
- **통계 분석**: "데이터베이스에 등록된 전체 매출 건수는 몇 건인가요?"

---

## 2. 비정형 데이터 검증 (문서/Vector 검색)

`docs/` 폴더에 업로드된 가이드라인과 규정 내용을 기반으로 답변하는지 확인합니다.

- **연차 규정**: "회사의 연차 휴가 부여 기준에 대해 설명해줘."
- **보안 지침**: "사내 보안 사고 예방을 위한 핵심 지침 3가지를 알려주세요."
- **복리후생**: "우리 회사의 주요 복리후생 제도에는 어떤 것들이 있나요?"

---

## 3. 하이브리드 검색 검증 (SQL + Vector 통합)

문서의 정책과 시스템의 실시간 데이터를 조합하여 지능적인 답변을 내놓는지 확인합니다.

- **정책 + 개인 조회**: "**박지은님**의 남은 휴가 일수를 알려주고, 이 결과가 사내 **연차 정책**에 맞게 산정되었는지 설명해줘."
- **부서 + 실적 통합**: "개발팀 직원들의 목록을 보여주고, 최근 **개발팀 관련 매출**이 있다면 함께 요약해줘."
- **종합 분석**: "현재 우리 회사의 **보안 가이드**를 요약해주고, 현재 보안팀에 소속된 직원이 누구인지 알려줘."

---

## 💡 테스트 팁

1. **모델 성능**: `deepseek-r1` 모델은 사고 과정(CoT)이 포함되어 답변이 다소 느릴 수 있습니다. 인내심을 갖고 기다리면 더 정확한 답변을 얻을 수 있습니다.
2. **데이터 업데이트**: 새로운 파일을 `docs/`에 넣은 후에는 반드시 [인제스트 관리] 페이지에서 **MD 변환**과 **벡터 임베딩**을 완료해야 검색이 가능합니다.
