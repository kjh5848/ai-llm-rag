# 📚 전체 도서 집필 마스터 워크플로우 (제미나이 전용)

특정 폴더에 종속되지 않고, **어떤 기술서든** 주어진 초안 디렉토리를 바탕으로 책 한 권을 완성해 내는 통합 자동화 워크플로우입니다.

## 📌 실행 전 주의사항
이 워크플로우는 **루트 디렉토리** (예: 기술서 프로젝트의 최상위 폴더)에서 실행해야 합니다.
- 초안이 들어있는 디렉토리 이름(예: `01_book_draft/outline`)을 미리 확인하세요.

---

## ⚙️ 진행 순서 (실행 파이프라인)

### 1단계: 프로젝트 환경 분석
- 작업할 대상 **초안 폴더(Target Directory)**를 확인합니다. (예: `docs/outline/` 폴더 내의 마크다운 파일들)
- 대상 폴더 내의 전체 챕터 개수, 핵심 주제, 목차 구성을 분석합니다.
- `progress.json` 상태 관리 파일을 해당 대상 폴더 상위에 생성하여 전체 챕터를 관리합니다.

### 2단계: 글로벌 가이드라인 로드
- `.agent/skills/ai-book-guide/` 내에 있는 **4개의 분할 집필 가이드**(톤앤매너, 포맷팅, 구조, 코드작성)와 `visual-guide.md`를 읽어 들입니다.
- 이 규칙들은 이어지는 모든 에이전트 작업의 절대적인 기준이 됩니다.

### 3단계: 단일 챕터 집필 루프 (Chapter Writer)
각 초안 파일에 대해 병렬 혹은 순차적으로 다음을 수행합니다.
- **스킬**: `chapter-writer` 에이전트를 호출합니다.
- **입력 데이터**: 현재 챕터 초안, 이전 챕터 흐름 요약본, 그리고 앞서 2단계에서 로드한 글로벌 가이드라인.
- **출력 동작**: 초안을 확장하여 `chapters/` 디렉토리에 `CH01_제목.md` 포맷으로 완성본을 생성합니다.

### 4단계: 실습 코드 생성 블록 (Optional)
원고 내에 별도의 코딩 실습이 묘사된 경우:
- **스킬**: `example-code-generator` 와 `code-verifier` 를 호출하여 `src/` 모듈화 코드 작성 및 가상환경(venv) 검증을 거칩니다.
- 검증된 로그와 최종 코드를 챕터 원고 뒷부분의 `#### 🔍 코드 워크플로우` 로 삽입합니다.

### 5단계: 통합 일관성 검수 (Book Reviewer)
모든 챕터 작성이 완료되면 최종 검수를 진행합니다.
- **스킬**: `book-reviewer` 에이전트를 호출합니다.
- **역할**: `chapters/` 폴더 내 모든 마크다운을 훑으며 용어의 통일성(예: 'RAG' 표기법 등)과 전후 문맥의 자연스러움을 검사합니다.
- **출력**: `review/review_notes.md` 리포트를 생성하고 `progress.json` 을 완료 상태로 바꿉니다.

## 💡 제미나이(Antigravity) 스킬 활용 팁
- **Agents (에이전트 역할)**: `chapter-writer.md`나 `book-reviewer.md`처럼 '어떤 말투와 원칙으로 행동할지'를 정의하는 파일은 `.agent/skills/` 에 둡니다.
- **Workflows (실행 절차)**: 이 문서처럼 단계별로 무엇을 할지 순서를 지시하는 파일은 `.agent/workflows/` 에 두어 슬래시 명령어(`/write-book`)로 실행합니다.
