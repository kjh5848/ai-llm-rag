# ✍️ Chapter Writer (챕터 집필기)

기술 도서의 단일 챕터를 작성하는 스킬입니다. 반드시 `technical-book-guide` 의 가이드라인을 준수해야 합니다.

## 📌 주요 임무
1. **초안 확장**: 제공된 아웃라인(초안)과 이전 챕터의 맥락을 바탕으로 상세 본문을 작성합니다.
2. **톤앤매너 적용**: '하십시오'체를 사용하며, 멘토가 설명하는 듯한 권위있고 친절한 톤을 유지합니다.
3. **구조화**: 개념 설명 -> 코드 실습 -> 코드 워크플로우(해설) -> 챕터 마무리 순서를 따릅니다.
4. **시각 자료 배치**: `visual-guide.md` 에 따라 적절한 위치에 16:9 비율의 이미지 프롬프트 또는 다이어그램(Mermaid)을 삽입합니다.

## ⚠️ 필수 준수 규칙
- 마크다운 볼드체 사용 시, 괄호나 따옴표로 끝나는 강조 구문 뒤에는 반드시 공백을 둡니다. (예: `**함수(Function)** 를`)
- 코드를 설명할 때는 라인 단위의 지루한 나열을 피하고, 모듈 간의 상호작용과 아키텍처 관점에서 설명합니다.
- 마지막 섹션은 반드시 `## X. 정리하며` 로 구성하여 챕터의 핵심을 요약합니다.
- 다이어그램 작성 시 괄호 ()를 사용할 때는 텍스트 전체를 쌍따옴표 ""로 감싸야 합니다.
