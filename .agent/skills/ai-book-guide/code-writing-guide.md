# 💻 실습 및 코드 작성 가이드 (Code Writing System)

독자가 에러 없이 코드를 읽고 따라칠 수 있도록 돕는 실무 코드 작성 지침입니다.

## 1. 코드 작성 표준
- **네이밍 규칙**: 변수/함수는 `snake_case`, 클래스는 `PascalCase` 사용.
- **주석 및 힌트**: 모든 함수에 통일된 한글 Docstring과 타입 힌트(Type Hinting) 적용 강제.
- **모듈화**: 하나의 긴 코드 뭉치 대신 `src/`, `config/` 등 실무 구조 적용.

## 2. 코드 설명 표준 (Code Explanation Standard)
코드를 단순히 던져놓고 넘어가는 것을 엄격히 금지합니다.
- **헤더 규칙**: 중요 코드 블록 하단엔 반드시 **`#### 🔍 코드 워크플로우 (Code Workflow)`** 섹션 배치.
- **내용 구성**: 논리 순서에 따라 숫자를 매겨 해설합니다 (`1.`, `2.`).
  - **입력(Input)**: 데이터의 시작
  - **처리(Process)**: 로직 요약
  - **출력(Output)**: 최종 리턴 혹은 생성되는 파일
- **코드 전체 제공**: 생략하지 말고 전체 코드를 보여주어 독자의 복붙(Copy & Paste) 편의 도모.

## 3. 실습 진행 규칙 (Actionable Steps)
- **실습 폴더 이동**: 터미널 명령어를 줄 때 `cd ..` 대신 "~ 디렉토리로 이동하세요"와 같은 행동 지시 선호.
- **코드 로직 박스**: 실습 단계마다 알고리즘/정규식 등 어려운 부분에 `#### 🔍 코드는 어떻게 작동하나요?` 해설 추가.
- **OS 구분 분기**: 패키지 설치나 키 등록 등 윈도우/맥OS의 차이는 명확하게 분리 안내.
- **이중 언어 멀티모달 실습**: LLaVA 등 AI 정제 실습은 'PDF Text 추출' ➔ 'AI 정제'의 2단계 파이프라인을 강조합니다.

## 4. 실습 환경 구성 원칙
- **의존성 최소화**: `requirements.txt` 로 패키지 설치 한방에 끝내기. (가상환경 활용 안내 필수)
- **데이터 로드 보장**: 구하려고 헤맬 필요 없이, 실습용 PDF/이미지를 저장소 내에 함께 제공.
- **결과 검증 포인트**: '이렇게 나오면 성공입니다'라는 문자열 결과 캡처 혹은 스크린샷 제시.
