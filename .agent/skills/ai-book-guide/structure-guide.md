# 🏢 원고 구조 가이드 (Structure System)

각 챕터가 독자 중심의 학습 곡선을 그릴 수 있도록 체계적인 구조를 정의합니다.

## 1. 3단계 핵심 전략 (Learning Curve)
이론과 실습의 황금비율을 맞추기 위해 3단계 구성을 준수합니다.
1. **[개념 이해]**: Mermaid 도식을 활용하여 아키텍처나 원리를 먼저 설명.
2. **[따라하기]**: 명확한 코드 블록과 라인별 주석 포함하여 단계별 실습 안내.
3. **[원리 분석]**: "실무에선 이렇게 막아야지" 등 심화/예외 처리/최적화 원리 분석.

## 2. 원고 구조 템플릿
모든 챕터는 다음 순서로 구성되어야 합니다:
1. **헤더 및 요약**: 단원의 목적을 1~2문장으로 요약.
2. **본문 (다이어그램 + 실습 + 코드 분석)**
3. **도구 소개 포맷**(신규 툴 소개 시): `도구명`, `기능`, `입력 스키마`
4. **마무리 및 연결**: 학습 내용 요약 및 다음 장 연계성 제시.

## 3. 챕터 마무리 규칙 (Conclusion Rule)
각 장의 마지막엔 독자가 배운 것을 확실히 챙겨갈 수 있도록 돕습니다.
- **섹션 제목**: 각 장의 마지막 섹션은 반드시 **"정리하며"** 로 통일 (예: `## 3. 정리하며`).
- **개조식(Bullet Points)**: 줄글로 나열하지 말고 핵심 내용을 불렛 포인트로 정리.
- **두괄식(Deductive)**: 각 항목 첫 문장에 결론을 제시하고 부연설명은 뒤로 뺍니다.
